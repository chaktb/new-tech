Auto-Posting 원리와 방법

매일 아침 자동으로 논문·기사를 수집·요약·게시하는 파이프라인

← Home

이 사이트는 매일 아침 08:00 (KST) 에 Si Photonics · PIC · Quantum Computing 분야의 새 논문과 기사를 자동으로 수집하고, 로컬 LLM 으로 한국어 요약을 붙여 새 페이지로 게시합니다. 사람이 손대지 않아도 매일 콘텐츠가 갱신됩니다. 아래는 그 동작 원리와 구축 방법 정리입니다.

1.전체 아키텍처

핵심 아이디어는 실행 주체를 클라우드가 아니라 로컬 서버(DGX Spark)에 둔다는 점입니다. LLM(Ollama)이 로컬에 있고, 사설망 안에 있어 외부 클라우드에서 접근이 어렵기 때문입니다. Cloudflare 는 단순히 정적 배포만 담당합니다.

2.왜 이런 구조인가

처음에는 "Cloudflare Cron 이 매일 8시에 실행"하는 방식을 고려했지만, 두 가지 제약이 있었습니다.

  • Ollama 접근 불가 — LLM 서버가 사설망(Tailscale, 100.x 대역) 안에 있어 Cloudflare 엣지에서 직접 호출할 수 없습니다.
  • 토큰 노출 위험 — GitHub 에 매일 커밋하려면 클라우드 함수에 push 토큰을 심어야 합니다.

그래서 항상 켜져 있고, Ollama 와 git 이 모두 로컬에 있는 DGX Spark 에서 전 과정을 실행하고, Cloudflare 는 push 를 감지해 정적 배포만 하도록 역할을 나눴습니다.

3.동작 단계

Step 1 · 수집 (Fetch)

arXiv API 의 physics.optics, quant-ph, physics.app-ph 카테고리를 최신순으로 조회하고, Phys.org 의 optics-photonics / quantum-physics RSS 와 Nature Photonics 현재 이슈 RSS 를 함께 파싱합니다.

Step 2 · 필터 (Filter)

제목·초록을 키워드 정규식(silicon photonic, PIC, quantum comput, LNOI 등)으로 걸러 관련 항목만 남기고, 최근 30시간 이내·중복 제거 후 상위 N건을 선정합니다.

Step 3 · 요약 (Summarize)

각 항목을 로컬 Ollama(gpt-oss:120b)의 /api/generate 로 보내 한국어 3문장 이내 요약을 생성합니다. 외부 API 키·비용이 들지 않습니다.

Step 4 · 게시 (Publish)

public/posts/YYYY-MM-DD.html 를 생성하고, public/index.html 의 <!-- AUTO_CARDS --> 마커 아래에 카드 링크를 자동 삽입합니다.

Step 5 · 배포 (Deploy)

변경분을 GitHub 에 git push 하면, Cloudflare Workers 가 이를 감지해 npx wrangler deploy 로 ./public 을 정적 자산으로 배포합니다. 1~3분 뒤 사이트에 반영됩니다.

4.기술 스택

Python (urllib · xml) arXiv API RSS (Phys.org) Nature Photonics Ollama · gpt-oss:120b Git / GitHub Cloudflare Workers wrangler [assets] crontab

5.구축 방법 (재현 절차)

5-1 · Cloudflare 정적 배포 설정

Workers 프로젝트가 정적 파일 디렉토리를 인식하도록 리포 루트에 wrangler.toml 을 둡니다.

name = "new-tech"
compatibility_date = "2026-07-08"

[assets]
directory = "./public"
주의: 이 설정이 없으면 배포 시 Could not detect a directory containing static files 에러가 납니다. 정적 파일은 반드시 ./public 아래에 두어야 합니다.

5-2 · 수집·요약·게시 스크립트

scripts/daily_photonics_digest.py 하나가 수집→필터→요약→HTML→push 를 모두 처리합니다. 상단 CONFIG 블록에서 소스·모델·경로를 조정합니다.

OLLAMA_URL   = "http://localhost:11434"
OLLAMA_MODEL = "gpt-oss:120b"
REPO_DIR     = "~/new-tech"
MAX_ITEMS    = 8
ARXIV_QUERIES = [
    "cat:physics.optics AND (silicon photonics OR photonic integrated)",
    "cat:quant-ph AND (photonic OR integrated OR quantum computing)",
    "cat:physics.app-ph AND (silicon photonics OR PIC)",
]

5-3 · 매일 실행 (crontab)

DGX Spark 의 시스템 시간대가 KST 이므로 매일 08:00 에 실행합니다.

PATH=/usr/bin:/usr/local/bin:/bin
0 8 * * * /usr/bin/python3 $HOME/new-tech/scripts/daily_photonics_digest.py \
    >> $HOME/new-tech/digest.log 2>&1
Tip: cron 은 대화형 셸과 환경이 달라 경로를 못 찾을 수 있으므로 PATH 줄과 절대경로(/usr/bin/python3)를 함께 지정하는 것이 안전합니다.
